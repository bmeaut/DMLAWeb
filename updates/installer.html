The DMLA Launcher and installer

The DMLA launcher (dmla.exe) is responsible for installing, updating and managing components of the DMLA tooling, and starting the execution/validation of D# files/workspaces.

Getting started

To get started, download the dmla.exe executable, and start it without any arguments - this will start the installation process.

The DMLA toolset consists of 3 components:

  • the launcher (dmla.exe)
  • a docker image used by the dockerized runtime (dmlateam/dmla-prod)
  • a VS Code extension that provides editor features

The installer prompts you to install all dependencies, and installs all the required components.

Every subcommand of the dmla command supports the -h or--help flags that display a help menu specific to the subcommand.

Running code

To run code, use the dmla run command, or the start options provided by the VS Code extension.

The dmla run command accepts the following arguments:

  • --nobootstrap Turns off the bundled bootstrap.
  • --dap <PORT> Starts debugging, waits for a debugger to be attached to the specified port.
  • --source <FILES>... Specifies the domain files (or directories) to be loaded.
  • --entrypoint <ENTRYPOINT> Specifies the operation to start.
  • --showcmd Displays the container launch command before launching it.
  • --useprofilinginstrument Turns on DMLA profiling.

Examples

To validate a workspace, one can simply run dmla run ..

To start a specific operation (in this example WarehousePackage.Init), use dmla run --source <FILES> --entrypoint WarehousePackage.Init

To turn on profiling, add the --useprofilinginstrument flag.
